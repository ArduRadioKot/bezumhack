# IDOR Attack Report

## 📋 Описание уязвимости

**IDOR (Insecure Direct Object Reference)** — уязвимость контроля доступа, возникающая когда приложение предоставляет прямой доступ к объектам по идентификатору без проверки прав пользователя.

---

## 🎯 Цели атаки

- Получить несанкционированный доступ к данным пользователей
- Извлечь информацию об устройствах
- Получить доступ к истории заказов
- Выявить данные администратора

---

## 🔧 Проведенные атаки

### Атака 1: Доступ к устройствам пользователя по email

```bash
# Запрос к API с email администратора
curl -sS "http://82.202.142.35:8080/api/users/admin@example.com/devices" | python3 -m json.tool
```

**Результат:** ✅ Успешно

Получены полные данные об устройстве:

- IP-адрес: `5.129.227.90`
- Browser: Chrome 144.0.0.0 (Yandex Browser 26.3)
- OS: Windows 10
- GPU: Intel UHD Graphics
- Screen: 1536x864
- Timezone: Europe/Moscow
- Canvas/Audio fingerprints
- И другие телеметрические данные

## 📊 Анализ уязвимостей

### Найденные уязвимые endpoints

| Endpoint                     | Метод | Уровень риска | Описание                               |
| ---------------------------- | ----- | ------------- | -------------------------------------- |
| `/api/users/{email}`         | GET   | 🔴 Критичный  | Раскрывает личные данные, баланс, роль |
| `/api/users/{email}/devices` | GET   | 🟠 Высокий    | Фингерпринтинг устройств, IP-адреса    |
| `/api/orders`                | GET   | 🔴 Критичный  | Доступ ко всем заказам без авторизации |

### Эксплуатируемые уязвимости

1. **Отсутствие авторизации** — endpoints не требуют токена/сессии
2. **Отсутствие проверки прав** — любой пользователь может получить данные другого
3. **Предсказуемые идентификаторы** — email используется как идентификатор
4. **Избыточные данные в ответах** — сервер возвращает чувствительную информацию

## 📝 Выводы
